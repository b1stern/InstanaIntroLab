## Introduction

This lab is an introduction into Instana and how Instana can be used to observe hybrid applications  The application consists 
of a cloud native component running in kubernetes (OpenShift) and a traditional server with IBM MQ and IBM App Connect Enterprise.  
Students will get an overview of Instana's key capabilities including how Instana can help them get to root cause quickly and in an intuitive way by
allowing the Instana UI to guide them through the diagnosis.  Students will explore Instana's capability to gather End User Experience (EUM) data, 
application performance, infrastructure monitoring, and more.


1) Students will explore key aspects of the Instana user interface.   

2) In the 2nd section of the lab, students will perform a diagnostic scenario in the hybrid application


After completing the Login section, use the other links to go directly to one of the lab exercises.


## Logging into the Environment

For the lab, you will be using the following Instana environment. Open your browser and login to https://instana-labs.instana.io/

Prior to the class, you should have received an invitation via e-mail from support@instana.io to join the Instana environment.  In the email, there is a button to **Join Instana**.
Click on the button.   If you already have an Instana google account, use that account to login.  Otherwise, create an account.

If you have difficulty setting up an instana account, you can login with  user account apmdemo-2@yahoo.com.  Your instructor can provide the password for the apmdemo-2@yahoo.com user account.

After successfully logging into the Instana environment, you should see the main landing page that includes a list of Websites & Mobile Applications, Applications, Platforms, Infrastructure, and Events.
